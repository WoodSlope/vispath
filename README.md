# VisPath · 视觉路径

VisPath 是一个可直接在浏览器中使用的 AI 视觉方向工作台。输入提示词或参考图后，可以生成多套提示词方案，提交图片生成，并按批次查看、筛选和对比结果。

## 在线使用

打开 GitHub Pages 页面即可使用，无需安装软件。当前版本仅支持宽度大于 `900px` 的电脑端浏览器。

页面由以下静态文件运行：

```text
index.html
styles.css
app.bundle.js
```

## API 配置

点击页面右上角的“API 配置”，分别填写：

- 文本 API：API 地址、API Key、模型。
- 生图 API：API 地址、API Key、模型、同步或异步生成方式。

文本和生图可以使用两个不同中转站。API 地址填写到 `/v1`，页面会自动拼接 `/responses`、`/images/generations`、`/images/generations/async` 和任务查询接口。

旧版共用 API 地址会自动回填到文本和生图配置；重新保存后会转为两个独立地址。

## 使用要求

- 中转站必须允许 GitHub Pages 域名发起跨域请求（CORS）。
- API Key 只保存在当前浏览器的 IndexedDB 中，不会写入 GitHub 文件。
- 浏览器请求仍会携带 Key，请使用可撤销、有限额的专用 Key，不要使用主账号 Key。
- 生成历史、任务 ID 和远程图片地址只保存在当前浏览器中。
- 默认请求 Base64 图片，避免结果依赖中转站返回的临时外链；若服务强制返回 URL，图片仍受该地址有效期、网络和 CORS 影响。

## 主要能力

- 输入提示词、上传或粘贴参考图。
- 生成 `2—6` 套提示词方向。
- 分别配置文本和生图 API。
- 支持同步及异步图片生成。
- 按提交批次浏览紧凑结果卡。
- 显示图片比例、实际尺寸和生成状态。
- 查看大图、下载原图和批次对比。
- 搜索、筛选、收藏、删除和撤销历史结果。
